## [πFS](https://github.com/philipl/pifs)
**Score:** 464 | **Comments:** 115 | **ID:** 48480978

> **Article:** πFS is a filesystem that stores data by encoding it as positions within the decimal digits of pi, eliminating the need for traditional disk storage. The project works by searching for each byte of a file within pi's digits and recording the offset where that byte sequence begins, allowing data to theoretically be reconstructed from pi alone. However, this approach faces fundamental limitations: the index required to locate data in pi typically exceeds the size of the original data, making practical storage inefficient. The concept serves primarily as an amusing thought experiment rather than a viable storage solution.
>
> **Discussion:** Commenters explored the mathematical and practical impossibilities of πFS, with mike_hock and adzm explaining that while infinite random sequences theoretically contain all possible data, the indexing overhead makes this approach no more efficient than direct storage. The discussion delved into whether pi is normal (containing all finite sequences) - a conjecture that remains unproven despite pi's known irrationality. Several users drew parallels to the Library of Babel and the debunked Sloot Digital Coding System, with jamwise noting that LLMs actually achieve effective compression through lossy methods. bobim's philosophical concern about pi containing all knowledge sparked debate about the unintuitive nature of infinity, while mzelling questioned the project's practical value, with tcoff91 confirming it's primarily for amusement. Technical analysis by Levitating and csunoser demonstrated why finding arbitrary byte sequences in pi requires exponentially more bits to encode the position than the data itself.
