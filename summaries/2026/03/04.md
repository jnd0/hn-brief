## [RFC 9849. TLS Encrypted Client Hello](https://www.rfc-editor.org/rfc/rfc9849.html)
**Score:** 242 | **Comments:** 118 | **ID:** 47244291

> **Article:** RFC 9849 standardizes TLS Encrypted Client Hello (ECH), a protocol that encrypts the Server Name Indication (SNI) field in the TLS handshake to prevent network observers from seeing which website a client is visiting. It evolved from the earlier ESNI draft and has seen adoption from major providers like Cloudflare and Firefox. A significant technical feature is that the server is not required to validate the "public name" presented in the outer handshake, allowing clients to use a censor-approved domain to connect to a different, hidden website. This specification aims to enhance privacy against SNI-based blocking and surveillance.
>
> **Discussion:** The discussion highlighted real-world use of precursor technologies like ESNI to bypass ISP-level blocks, such as in India with Jio, and excitement about ECH's potential for mainstream deployment. A key theme was ECH's role in censorship circumvention, particularly the "public name" feature allowing connections to hidden sites using approved domains, which sparked a debate about age verification laws and parental controls; one user argued it makes network-level filtering impossible, while another countered that device-level client software remains a viable solution. The impact on bot detection was debated, with one claim that ECH kills TLS fingerprinting for bots being corrected: while it prevents passive third-party (e.g., ISP) fingerprinting, servers like Cloudflare can still decrypt and fingerprint the inner ClientHello, creating a paradox for a company that both adopts ECH and sells bot detection. Technical nuances included whether IP-based certificates are allowed, with clarification that ECH is primarily beneficial for shared hosting/CDN scenarios where multiple domains share an IP, not for direct connections to a single IP address, and a correction that ECH only hides SNI from on-path observers, not from the server itself.
