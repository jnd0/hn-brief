## [Why does C have the best file API](https://maurycyz.com/misc/c_files/)
**Score:** 138 | **Comments:** 125 | **ID:** 47209788

> **Article:** The article argues that C's file API excels due to its ability to use mmap with structs to directly map binary data, a feature unique to C's low-level memory control. It highlights that C allows defining structs to correspond to binary formats per the C ABI on a system, enabling precise but risky memory access. The discussion notes that while C's approach is powerful, it risks undefined behavior if files change during mapping and lacks portability across architectures due to endianness and struct layout differences.
>
> **Discussion:** The discussion centers on whether C's mmap-based file handling is truly superior or a legacy artifact. Many argue that while C offers direct control, it introduces risks like undefined behavior from concurrent file modifications and requires manual handling of endianness and struct padding, which other languages abstract away. Critics counter that modern alternatives like io_uring or libraries (Cap’n Proto, FlatBuffers) provide safer, more efficient solutions without C's pitfalls. Some emphasize mmap's cross-language availability (e.g., Python, Java) and its role as a POSIX syscall, not exclusive to C. Debates also highlight historical context, with C's dominance in system programming shaping its perceived "best" status despite modern advancements. Community reactions range from nostalgia for C's simplicity to frustration over its error-prone nature in today's safer programming environments.
