## [New Nginx Exploit](https://github.com/DepthFirstDisclosures/Nginx-Rift)
**Score:** 411 | **Comments:** 96 | **ID:** 48138268

> **Article:** A new Nginx exploit called "Nginx Rift" (CVE-2026-42945) has been discovered, which is a heap buffer overflow in NGINX's ngx_http_rewrite_module that has existed since version 0.6.27 (2008). The exploit can lead to unauthenticated Remote Code Execution through specific configurations involving `rewrite` directives with question marks in replacement strings followed by `set` or `if` directives. The vulnerability affects approximately 39-43% of public-facing web servers since Nginx holds that market share. Patches have been released in version 1.31.0 and 1.30.1 for OSS, plus F5 Plus updates.
>
> **Discussion:** The discussion centers on the severity of the Nginx exploit and the role of ASLR in mitigating it. Security professionals emphasize that ASLR is a defense-in-depth technique that can be bypassed with sufficient skill and time, especially as LLM agents lower the barrier to exploitation, while others counter that ASLR can completely mitigate individual vulnerabilities without information leaks. Alternative web servers written in memory-safe languages like Caddy (Go) and Jetty (Java) were discussed, but noted to have their own vulnerabilities, with some arguing that memory safety doesn't guarantee security if implementation flaws exist. The conversation also touched on the widespread use of affected directives in Nginx configurations, with Ubuntu having patched the vulnerability while Debian lagged behind. Additionally, there was a side discussion about software versioning approaches, noting Nginx's long-term version 1.x versus newer projects like React's version 19.
