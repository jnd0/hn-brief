## [GeoJSON](https://geojson.org/)
**Score:** 145 | **Comments:** 68 | **ID:** 48060918

> **Article:** GeoJSON is a JSON‑based format for encoding geographic features, defined in RFC 7946, that standardizes points, lines, polygons, and collections with a focus on WGS84 coordinates. It is widely adopted for web mapping, enabling libraries like Leaflet and Mapbox to render spatial data directly from JSON. The format deliberately omits topology, making it simple but sometimes problematic for operations such as simplifying adjacent polygons, which can create slivers. Despite its simplicity, GeoJSON’s human‑readable structure has led to a rich ecosystem of tools, including binary alternatives like FlatGeobuf and Geobuf for more efficient storage and transmission.
>
> **Discussion:** The thread sparked a debate over GeoJSON’s suitability for complex spatial tasks. Some participants highlighted the format’s lack of topological awareness, noting that simplifying neighboring polygons independently can produce gaps or overlaps, and that TopoJSON offers a remedy by encoding shared boundaries once. Others defended GeoJSON, arguing that proper data preparation and the availability of tools like GeoJSON‑simplify mitigate these issues, and that the format’s readability and widespread tooling outweigh its drawbacks. A technical contention emerged around the efficiency of JSON parsing, with critics pointing out memory overhead and advocating for binary encodings, while proponents countered that gzip compression and the dominance of coordinate arrays reduce the impact. Discussions also touched on practical use cases, such as mapping warehouse robots or calculating delivery zones, where GeoJSON’s ease of use and TypeScript support proved advantageous, though edge cases like missing ZIP code boundaries prompted caution. Overall, the conversation balanced appreciation for GeoJSON’s accessibility against legitimate concerns about performance, topology, and data integrity in large‑scale geospatial applications.
