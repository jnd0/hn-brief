## [Deterministic Fully-Static Whole-Binary Translation Without Heuristics](https://arxiv.org/abs/2605.08419)

> **Article:** The paper presents "Elevator," a deterministic fully-static whole-binary translation system that sidesteps the code-versus-data determination problem through superset disassembly. Rather than trying to distinguish between code and data, Elevator interprets every executable byte offset as both data and a potential instruction sequence, building a superset control flow graph from all candidate decodes. The system resolves indirect jump targets at runtime through embedded lookup tables mapping original addresses to translated code addresses. Performance is claimed to be on par with or better than QEMU's user-mode JIT emulation, though the approach does not support self-modifying or JIT-compiled code.
>
> **Discussion:** Commenters immediately dove into the theoretical limitations, with dmitrygr invoking Rice's theorem to argue that adversarial code like dynamically constructed instruction buffers cannot be handled, though cooperative compiler-produced binaries work well. The certification angle for regulated industries like aviation and medical devices generated discussion about why static translation matters when JIT is prohibited, though camillomiller questioned the broader relevance of these industries to AI-driven disruption narratives.
