## [Deterministic Fully-Static Whole-Binary Translation Without Heuristics](https://arxiv.org/abs/2605.08419)
**Score:** 289 | **Comments:** 65 | **ID:** 48117810

> **Article:** The paper presents "Elevator," a deterministic fully-static binary translation system that converts x86_64 binaries to other architectures without using JIT compilation or heuristics. Unlike traditional approaches that struggle with the code-versus-data distinction, Elevator employs superset disassembly to simultaneously interpret every executable byte as both data and potential code instructions, building a comprehensive control flow graph. The system achieves performance on par with or better than QEMU's user-mode JIT emulation while producing a static, signable binary suitable for regulated industries where JIT is prohibited. However, it does not support self-modifying code or JIT-compiled code, and results in significant code bloat with a reported 50x expansion factor.
>
> **Discussion:** Commenters quickly identified both the technical innovation and fundamental limitations of the approach. Dmitrygr argued that Rice's theorem still applies, making perfect translation impossible for adversarial code like runtime-generated shellcode, though fsmv noted that Elevator's superset disassembly technique addresses this by translating everything as code. The certification angle emerged as particularly valuable for regulated industries like aviation and medical devices, where fizza_pizza explained that JIT compilation is prohibited because the running code must match what was certified. Performance comparisons revealed that QEMU's JIT is considered an easy target to beat, with da-x claiming their custom JIT achieved 2x-5x overhead versus QEMU's 10x-50x, while pm215 attributed QEMU's sluggishness to its multi-architecture design that prevents optimization for specific guest-host pairs. The discussion also covered why self-modifying code is increasingly rare and problematic on modern processors, and why supporting only x86_64 rather than 32-bit programs limits applicability for legacy software like old games.
