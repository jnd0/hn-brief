## [Bijou64: A variable-length integer encoding](https://www.inkandswitch.com/tangents/bijou64/)
**Score:** 198 | **Comments:** 72 | **ID:** 48323992

> **Article:** Bijou64 is a variable-length integer encoding designed to ensure every number has exactly one canonical representation, avoiding the ambiguity found in LEB128. It uses a prefix system where values from 0x0 to 0xF7 are single-byte integers, while values from 0xF8 to 0xFF use the upper bits as a counter for subsequent bytes. This approach allows the total length of the integer to be determined immediately upon reading the first byte, supporting magnitudes up to nearly 2^264.
>
> **Discussion:** Debaters focused heavily on the trade-offs between canonicality and performance, particularly regarding SIMD optimization. Some argued that Bijou64's lack of a fixed-position signal bit makes it slower than ULEB128 for parallel processing because it requires a serial walk to find value boundaries. Others defended the encoding's utility for storage and transmission, where dense packing and canonicality are critical for security and content-addressing to prevent attacks like signature forgery. A technical deep dive revealed that non-canonical LEB128 is actually useful in linking processes for DWARF and WASM to avoid shifting code when reference widths change. Some critics questioned the overall value of varints, suggesting that modern compression like zstd or flatter binary formats often outperform them in both speed and ergonomics.
