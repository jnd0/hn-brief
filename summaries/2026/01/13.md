**Score:** 99 | **Comments:** 43 | **ID:** 46588905

> **Project:** Agent-of-empires is a terminal-based session manager for AI coding agents, specifically designed for OpenCode and Claude Code. It acts as a dashboard to monitor multiple parallel agent sessions, leveraging `tmux` under the hood to manage and track the state of each agent (e.g., waiting for user input, working). Key features include managing multiple agents in one place, git worktree support for isolating parallel tasks on the same codebase, and a lightweight, terminal-native workflow inspired by the game *Age of Empires*.
>
> **Discussion:** The discussion centered on the project's role in the emerging field of AI agent orchestration. The author, river_otter, positioned the tool as a lightweight, terminal-first solution for managing a small fleet of coding agents, contrasting it with more GUI-heavy alternatives like conductor.build. A key technical point was the use of `tmux` capture-pane for status detection, which was noted as a simple and effective "KISS" approach, though some users pointed out that Claude Code's native hooks could offer a more elegant integration.

Users shared their own similar projects, highlighting a convergent evolution of ideas around managing multiple agents. There was significant interest in using such tools to facilitate experimentation with local LLMs (e.g., Ollama), as session management provides the necessary patience and oversight for less reliable models. Practical use cases were discussed, such as remote monitoring of agents from a phone. The conversation also touched on the broader trend of agent orchestration, referencing Steve Yegge's recent posts, and identified sandboxing as a potential future feature.
