## [Finding and fixing Ghostty's largest memory leak](https://mitchellh.com/writing/ghostty-memory-leak-fix)
**Score:** 465 | **Comments:** 103 | **ID:** 46568794

> **Article:** This article by Ghostty creator Mitchell Hashimoto details the process of finding and fixing a significant memory leak in the terminal emulator's scrollback buffer. The leak was caused by a mismatch between the standard memory size of scrollback "pages" and the non-standard sizes created by certain applications, such as Claude Code. When a non-standard page was allocated, the memory pool's recycling logic would incorrectly try to reuse it for a standard-sized page, leading to memory corruption and a leak. The fix was simple: if the oldest page in the pool is of a non-standard size, it is discarded rather than being recycled. Hashimoto also discusses the use of AI to generate visualizations for the blog post, valuing it for creating disposable, one-off diagrams.
