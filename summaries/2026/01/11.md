## [Finding and fixing Ghostty's largest memory leak](https://mitchellh.com/writing/ghostty-memory-leak-fix)
**Score:** 565 | **Comments:** 123 | **ID:** 46568794

> **Article:** This article by Ghostty creator Mitchell Hashimoto details the process of finding and fixing a significant memory leak in the terminal emulator. The leak was located in the `PageList`, a data structure responsible for managing scrollback history. The bug occurred when handling "non-standard" page sizes (e.g., lines with more characters than the typical terminal width).

The leak was triggered when the oldest page in the scrollback buffer was a non-standard size and needed to be pruned to make room for a new page. The code failed to correctly free the memory of the old page, causing it to be lost. The fix was simple: explicitly check if the page being pruned is non-standard and ensure it is freed. Hashimoto notes that while the bug existed for years, its occurrence was recently amplified by the use of tools like Claude Code, which often generate output that creates these non-standard lines. The article includes helpful visualizations to explain the memory management logic.
