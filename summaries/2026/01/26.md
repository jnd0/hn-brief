## [Oneplus phone update introduces hardware anti-rollback](https://consumerrights.wiki/w/Oneplus_phone_update_introduces_hardware_anti-rollback)
**Score:** 303 | **Comments:** 128 | **ID:** 46757944

> **Article:** OnePlus has rolled out a software update that activates a hardware anti-rollback mechanism on certain phone models. This mechanism uses eFuses—tiny, one-time programmable memory cells within the processor—to permanently record the version of the firmware currently installed. If a user attempts to downgrade to an older firmware version, the bootloader detects that the version number is lower than the value stored in the fuse and refuses to boot. This effectively prevents users from installing older versions of the operating system, including custom ROMs that are built against older firmware components.
>
> **Discussion:** The Hacker News discussion surrounding the OnePlus update centers on the motivations behind the move, the technical implementation, and the broader implications for consumer rights. Commenters are largely critical, viewing this as another step in OnePlus's decline from an enthusiast-friendly brand to a standard manufacturer focused on control and security compliance.

The primary debate revolves around why OnePlus would implement such a restriction. Several theories were proposed:
*   **Security and Theft Prevention:** The most cited reason is that a vulnerability was discovered allowing attackers to bypass theft locks by downgrading the firmware to an exploitable older version. By preventing downgrades, OnePlus ensures that stolen devices remain unusable and that the phones comply with carrier and regulatory requirements for remote disablement.
*   **Anti-Custom ROM Measures:** Skeptics argue that this is a deliberate move to kill the custom ROM community, forcing users onto official firmware. This locks users into OnePlus's software decisions, even if an update degrades performance, thereby encouraging the purchase of new devices.
*   **DRM and Compliance:** Some speculate it relates to enforcing DRM standards (like for Netflix) that break on older software versions.

Technically, users clarified that this is not a new concept. eFuses are standard in modern CPUs (including Qualcomm and Apple silicon) and have been used for years to store cryptographic keys, IMEI data, and secure boot versions. The mechanism works by comparing the firmware's embedded version number against the "blown" fuse value in the processor's ROM. If the firmware is older, the boot process halts. Commenters noted that Apple uses a similar, non-hardware approach via signature enforcement (SHSH blobs), which achieves the same result of preventing downgrades without physically altering the hardware.

The discussion expanded into a philosophical debate on ownership and the "Right to Repair." Many commenters argued that if a manufacturer can remotely alter a device's capabilities or prevent the installation of software of the user's choice, the user does not truly own the device. This sentiment was compared to modern automobiles, where manufacturers retain significant control over the vehicle's software, challenging the traditional definition of ownership.
