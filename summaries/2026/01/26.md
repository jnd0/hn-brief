## [Oneplus phone update introduces hardware anti-rollback](https://consumerrights.wiki/w/Oneplus_phone_update_introduces_hardware_anti-rollback)
**Score:** 430 | **Comments:** 255 | **ID:** 46757944

> **Article:** The linked article details how a recent OnePlus software update introduced a hardware-based anti-rollback mechanism. This mechanism uses "eFuses"—one-time programmable memory bits within the phone's processor—to permanently record the current firmware version. When the device boots, the bootloader checks the firmware version against the value stored in these fuses. If a user attempts to install an older firmware version (such as a custom ROM built on an older base), the boot process is rejected. This prevents users from downgrading their device's software, effectively locking the hardware to the most recent version it has booted.
>
> **Discussion:** The Hacker News discussion centered on the motivations behind OnePlus's decision, the technical implementation of eFuses, and the broader implications for device ownership.

A primary theme was the debate over OnePlus's intent. While some users cynically viewed the move as a way to force obsolescence and drive new sales, most technical explanations suggested a security-driven motive. Commenters pointed to a specific vulnerability that allowed attackers to bypass theft-detection locks by downgrading to an older, unpatched firmware. In this view, the anti-rollback feature is a security measure to prevent such exploits, similar to practices used by Samsung (for Knox) and Apple (which uses software signatures to prevent downgrades).

The technical discussion clarified how these mechanisms work. Users explained that eFuses are common in modern CPUs, used to securely store version numbers and cryptographic keys. The anti-rollback check occurs at a low level in the boot process, before the main operating system loads, making it difficult to bypass without physical hardware exploits. This prevents custom ROM developers from flashing older firmware, forcing them to build their ROMs against the latest vendor blobs.

Underlying the entire conversation was a strong sentiment about digital ownership and the "right to repair." Many commenters argued that by implementing irreversible hardware changes via software updates, manufacturers are undermining the concept of true ownership. The consensus was that while these features may serve security purposes, they ultimately limit user autonomy, restrict the ability to run custom software, and contribute to a culture where consumers are merely licensing the hardware they purchased.
