>
> **Discussion:** The Hacker News discussion centered on the common but often frustrating practice of software (OSes, drivers) using executable filenames to alter behavior.

Key points include:
*   **Widespread Practice:** Commenters noted this is not unique to Microsoft; graphics drivers and other low-level software often use filename-based heuristics for performance tweaks or compatibility hacks.
*   **Backward Compatibility:** Many defended Microsoft's approach, citing legendary examples (like the SimCity memory bug fix in Windows 95) where such hacks are essential for ensuring old software continues to work, which is a core tenet of Windows' success.
*   **Developer Frustration:** Others sympathized with the author's pain, highlighting the difficulty of debugging such obscure, non-obvious issues.
*   **Broader Context:** The discussion clarified that this is a side effect of Microsoft's "Agility SDK" strategy, where DirectX features are decoupled from the OS and applied via runtime patching, which can lead to unforeseen edge cases.
