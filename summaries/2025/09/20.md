## [Images over DNS](https://dgl.cx/2025/09/images-over-dns)
**Score:** 201 | **Comments:** 50 | **ID:** 45312515

> **Article:** The article "Images over DNS" by dgl demonstrates a proof-of-concept for transmitting and displaying images using only DNS queries. The technique involves encoding image data into a very long TXT record (or multiple records) and using a client-side script to query the DNS record and render the data as an image. The author notes that this method is slow and primarily a technical curiosity, but highlights that DNS is often allowed through firewalls, making it a potential covert channel.
>
> **Discussion:** The Hacker News discussion focused on the technical feasibility, limitations, and security implications of using DNS for data transport.

Key points include:
*   **Technical Limits:** Users debated the maximum size of DNS responses. The consensus is that a single DNS message is limited to ~64KB due to a 16-bit length field in the TCP header, but larger data can be sent by chaining multiple responses (similar to AXFR zone transfers).
*   **Caching Issues:** A user noted that DNS caching can cause reordering of records, which breaks time-sensitive applications like video streaming. The author confirmed this is why they used a single long record rather than multiple smaller ones.
*   **Security & Abuse:** Several comments highlighted that DNS tunneling is a well-known method for data exfiltration and bypassing firewalls. Corporate environments often restrict DNS traffic to internal resolvers to prevent this. One user shared an anecdote about customers abusing free DNS traffic to avoid data transfer fees, causing network-wide issues.
*   **Related Projects:** Users shared similar projects, including playing "Bad Apple" and "Doom" over DNS, a Chrome extension for a DNS-based CDN (cdn53), and Iodine, a tool for tunneling IPv4 over DNS.
*   **Image Format:** A minor technical point clarified that the image format used is HEIF containing AVIF data, not just AVIF.
