This is a long-awaited change that moves Firefox's configuration and data files from the default `~/.mozilla` directory to the standardized locations defined by the spec, such as `~/.config`, `~/.cache`, and `~/.local/share`. The move aims to reduce clutter in the user's home directory and align Firefox with other modern Linux applications.
>
> **Discussion:** The Hacker News discussion is a mix of celebration, skepticism, and philosophical debate, with the consensus being that while the move is a positive step, it comes after an absurdly long 21-year development cycle.

Key points of agreement and insight include:
*   **A Monumental Wait:** The most common sentiment is disbelief at the time it took to close the original bug report from 2003. Comments like "it only took 21 years" and references to "Gimp 3 levels of time" highlight the community's frustration with Mozilla's glacial pace on this issue.
*   **Broader Impact:** Users see this as a significant win for the Linux ecosystem. Firefox's adoption of the standard is expected to pressure other major non-compliant applications to follow suit, improving overall system consistency.

The primary disagreements and points of contention revolve around the implementation details and the spec itself:
*   **Incomplete Implementation:** A critical technical concern is raised about the patch. It appears to simply move the entire `.mozilla` directory to `.config/mozilla` rather than properly splitting files by type (configuration in `XDG_CONFIG_HOME`, cache in `XDG_CACHE_HOME`, and data in `XDG_DATA_HOME`). This is seen as a half-measure that misses the full benefit of the spec.
*   **Migration Path:** There is significant anxiety about how existing users' profiles will be handled. The discussion questions whether there is an automatic migration or if users will be left with broken profiles unless they manually set environment variables, a common pain point with such changes.
*   **The Value of the Spec Itself:** A minor debate emerges on the merit of the XDG spec. While most champion it for reducing dotfile clutter and respecting user-configured environments, a dissenting voice dismisses it as "petty," arguing that the location of config files is a trivial issue.
